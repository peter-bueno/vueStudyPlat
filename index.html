Reactivity Fundamentals

Understanding Vue's reactivity system

ref() vs reactive()

ref()

  • For primitive values
  • Accessed via .value
  • Can hold any type
  • Returns a reactive reference

reactive()

  • For objects and arrays
  • Direct property access
  • Only objects/arrays
  • Deep reactivity by default

Interactive Example

Count: 0

Name: John

import { ref, reactive } from 'vue'

// Using ref for primitives
const count = ref(0)
const increment = () => count.value++

// Using reactive for objects
const user = reactive({
  name: 'John',
  age: 30
})

const updateName = () => {
  user.name = user.name === 'John' ? 'Jane' : 'John'
}

Composition API vs Options API

Understanding both approaches

Options API Approach

‚úÖ Pros

  • Familiar structure
  • Easy to learn
  • Good for simple components
  • Clear separation of concerns

‚ùå Cons

  • Code fragmentation in large components
  • Limited reusability
  • Harder to compose logic
<template>
  <div>
    <h1>{{ title }}</h1>
    <button @click="increment">Count: {{ count }}</button>
  </div>
</template>

<script>
export default {
  data() {
    return {
      title: 'My Component',
      count: 0
    }
  },
  methods: {
    increment() {
      this.count++
    }
  },
  mounted() {
    console.log('Component mounted')
  }
}
</script>

Composition API Approach

‚úÖ Pros

  • Better logic organization
  • Improved reusability
  • Better TypeScript support
  • Easier testing

‚ùå Cons

  • Steeper learning curve
  • More manual setup
  • Requires discipline
<template>
  <div>
    <h1>{{ title }}</h1>
    <button @click="increment">Count: {{ count }}</button>
  </div>
</template>

<script setup>
import { ref, onMounted } from 'vue'

const title = ref('My Component')
const count = ref(0)

const increment = () => {
  count.value++
}

onMounted(() => {
  console.log('Component mounted')
})
</script>

State Management with Pinia

Managing global application state

Setting up Pinia

# Install Pinia
npm install pinia
// main.js
import { createApp } from 'vue'
import { createPinia } from 'pinia'
import App from './App.vue'

const app = createApp(App)
app.use(createPinia())
app.mount('#app')

Creating a Store

// stores/user.js
import { defineStore } from 'pinia'

export const useUserStore = defineStore('user', {
  state: () => ({
    users: [],
    currentUser: null,
    loading: false
  }),
  
  getters: {
    getUserById: (state) => (id) => {
      return state.users.find(user => user.id === id)
    }
  },
  
  actions: {
    async fetchUsers() {
      this.loading = true
      try {
        const response = await fetch('/api/users')
        this.users = await response.json()
      } finally {
        this.loading = false
      }
    },
    
    setCurrentUser(user) {
      this.currentUser = user
    }
  }
})
